## Codebase Patterns
- Validate required webhook environment variables in a small config helper and fail fast during Nest provider construction.
- Keep Twilio controllers thin and build TwiML in services so callback and recovery logic stay testable.
- Register `express.urlencoded({ extended: false })` in Nest bootstrap before handling Twilio webhooks, because Twilio posts `application/x-www-form-urlencoded` payloads.
- Keep Twilio lead state behind a repository boundary so Airtable and in-memory persistence can share the same webhook-facing service contract.
- Use `phone:<normalized-number>` as the primary lead key and fall back to `unknown:<sid>` when webhook payloads are missing a caller number.
- Prefer `TwilioService` unit tests with the in-memory repository and mocked `global.fetch`; this covers TwiML, recovery decisions, and SMS threading without live Twilio/OpenAI credentials.
